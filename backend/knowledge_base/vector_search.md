# 向量搜索技术

## 什么是向量搜索

向量搜索（Vector Search）是一种基于高维向量相似度的检索技术。它将文本、图像等非结构化数据转换为数值向量，然后通过计算向量之间的距离或相似度来找到最相关的数据。

## 向量嵌入原理

### 文本 → 向量
嵌入模型（Embedding Model）将文本映射到高维向量空间：
- 语义相似的文本在向量空间中距离更近
- 不同语言但含义相同的文本也会有相近的向量

### 常用嵌入模型
| 模型 | 维度 | 特点 |
|------|------|------|
| nomic-embed-text | 768 | 开源，多语言 |
| text-embedding-ada-002 | 1536 | OpenAI，英文优秀 |
| bge-large-zh | 1024 | 中文优化 |
| dmeta-embedding-zh | 1024 | 中文优化 |

## 相似度度量

### 余弦相似度（Cosine Similarity）
- 计算两个向量夹角的余弦值
- 范围：-1 到 1，越接近 1 越相似
- 最常用于文本检索

### 欧氏距离（Euclidean Distance）
- 计算两个向量之间的直线距离
- 距离越小越相似

### 内积（Inner Product / Dot Product）
- 两个向量对应元素相乘后求和
- 值越大越相似

## 向量数据库

### OpenSearch
- 基于 Elasticsearch 的开源分支
- 支持 KNN 向量检索
- 内置 HNSW 索引算法
- 支持混合检索（向量 + 关键词）

### Milvus
- 专为向量搜索设计的数据库
- 支持多种索引类型
- 高性能，适合大规模数据

### Chroma
- 轻量级向量数据库
- 适合开发和原型
- Python 原生支持

## 索引算法

### HNSW（Hierarchical Navigable Small World）
- 多层图结构
- 搜索速度快，精度高
- 内存占用较大

### IVF（Inverted File Index）
- 将向量空间划分为多个聚类
- 搜索时只查询相关聚类
- 适合大规模数据

### PQ（Product Quantization）
- 将高维向量压缩为低维表示
- 大幅减少内存占用
- 精度有一定损失

## 应用场景

1. **语义搜索**：理解查询意图，返回语义相关的结果
2. **推荐系统**：基于用户行为向量推荐相似内容
3. **图像检索**：以图搜图，找到视觉相似的图片
4. **去重检测**：检测重复或高度相似的文档
5. **聚类分析**：将相似文档自动分组
