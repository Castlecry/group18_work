# 技术开发规范与指南

## 第一章 开发环境

### 1.1 基础工具
- **IDE**：推荐使用 VS Code 或 IntelliJ IDEA
- **版本控制**：Git，远程仓库使用 GitLab
- **包管理**：前端使用 npm/pnpm，后端使用 pip/poetry
- **容器化**：Docker + Docker Compose 用于本地开发环境

### 1.2 开发规范
- 分支命名：`feature/功能名`、`bugfix/问题描述`、`hotfix/紧急修复`
- Commit 规范：遵循 Conventional Commits（feat/fix/docs/style/refactor/test/chore）
- Code Review：所有代码合并到主分支前必须经过至少 1 人 Review
- 单元测试覆盖率要求：核心业务逻辑 ≥ 80%

---

## 第二章 前端开发规范

### 2.1 技术栈
- 框架：Vue 3 + TypeScript
- 状态管理：Pinia
- UI 组件库：Element Plus
- 构建工具：Vite
- 路由：Vue Router 4

### 2.2 目录结构
```
src/
├── api/          # API 接口定义
├── assets/       # 静态资源
── components/   # 公共组件
├── composables/  # 组合式函数
├── router/       # 路由配置
├── stores/       # Pinia 状态管理
├── utils/        # 工具函数
├── views/        # 页面组件
└── App.vue       # 根组件
```

### 2.3 编码规范
- 组件命名：大驼峰（PascalCase），如 `UserProfile.vue`
- 变量/函数命名：小驼峰（camelCase）
- 常量命名：全大写 + 下划线（UPPER_SNAKE_CASE）
- CSS 使用 scoped 样式，避免全局污染
- 优先使用 Composition API（`<script setup>`）

## 第三章 后端开发规范

### 3.1 技术栈
- 框架：FastAPI（Python）
- 数据库：PostgreSQL + SQLAlchemy ORM
- 缓存：Redis
- 向量数据库：OpenSearch / pgvector
- 任务队列：Celery（异步任务）

### 3.2 API 设计规范
- RESTful 风格，URL 使用小写 + 短横线
- 统一响应格式：`{"code": 200, "message": "success", "data": {...}}`
- 错误处理：使用 HTTP 状态码 + 详细错误信息
- 分页参数：`page`（页码，从 1 开始）、`page_size`（每页条数，默认 20）
- 认证方式：JWT Token，通过 Authorization Header 传递

### 3.3 数据库规范
- 表名：小写 + 下划线（snake_case）
- 必须字段：`id`（主键）、`created_at`、`updated_at`
- 软删除：使用 `is_deleted` 布尔字段，不物理删除
- 索引：外键字段、常用查询字段必须建索引

## 第四章 部署与运维

### 4.1 部署流程
1. 代码合并到 `main` 分支
2. CI/CD 自动运行测试和构建
3. 部署到 staging 环境进行集成测试
4. 手动确认后部署到 production 环境

### 4.2 监控与告警
- 应用监控：Prometheus + Grafana
- 日志管理：ELK Stack（Elasticsearch + Logstash + Kibana）
- 告警渠道：企业微信群机器人 + 邮件
- 关键指标：API 响应时间、错误率、服务器 CPU/内存使用率

### 4.3 安全规范
- 敏感信息（密码、密钥）不得硬编码，使用环境变量或密钥管理服务
- API 接口必须做鉴权和权限校验
- 用户密码使用 bcrypt 加密存储
- 定期更新依赖包，修复已知安全漏洞
- SQL 查询使用参数化，防止注入攻击
